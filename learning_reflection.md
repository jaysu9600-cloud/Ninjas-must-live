# 《忍者必須活》：從零到一的 Python 遊戲開發自主學習心得

### 二、目的
本計畫旨在透過 Python 與 Pygame 框架，從零開始開發一款具備物理跳躍與碰撞檢測機制的橫向捲軸遊戲。除了學習基礎的物件導向程式設計（OOP），也嘗試體現「Vibe coding」的精神，將創意快速轉化為可執行的 Demo。

### 三、過程
學習過程分為四個階段：首先是環境建置與 Pygame 視窗初始化；接著設計 `Player` 類別，實作受重力影響的跳躍邏輯；第三階段是建立障礙物生成機制與隨時間增加難度的加速系統；最後則是關鍵的碰撞檢測與遊戲結束邏輯，確保遊戲的完整性與挑戰性。

### 四、困難與解決方法
在實作跳躍機制時，曾遇到角色座標與地面判定不準確，導致反覆彈跳或無法再次跳躍的問題。經研究後，我定義了明確了 `GROUND_Y` 常數，並在 `update` 函式中加入狀態鎖定 `is_jumping`，確保角色只有在接觸地面時才能觸發跳躍，並嚴格修正落地後的座標偏差。

### 五、心得
這是我首次嘗試完整的軟體開發流程，看著角色從一個靜止圖案變成能跳躍、避開障礙的忍者，成就感極大。過程中我不斷修正邏輯漏洞，體會到寫程式不只是打代碼，更是嚴謹的邏輯推敲。這次經驗開啟了我對資訊科學的興趣，未來希望能挑戰更高難度的演算法與特效。
