# 任務完成報告 (Score System Implementation)

## 任務摘要
在 `app.py` 中新增計分系統，實現每存活一秒增加分數，且每過十秒增加的分數額度會遞增的功能。

## 完成項目
1. **規格制定**：建立 `spec.md` 文件，詳細定義計分系統的邏輯與 UML 圖表（流程圖、循序圖、類別圖）。
2. **變數初始化**：在 `app.py` 中初始化計分變數（`score`, `score_rate`, `last_score_tick`, `last_rate_increase_tick`）與字型系統。
3. **計分邏輯實作**：
    - 利用 `pygame.time.get_ticks()` 追蹤時間。
    - 每 1000 毫秒（1 秒）將 `score` 增加 `score_rate`。
    - 每 10000 毫秒（10 秒）將 `score_rate` 增加 10。
4. **顯示系統**：在遊戲視窗左上角即時顯示當前分數與增量額度。
5. **文件更新**：
    - 修復並更新 `README.md`。
    - 同步更新 `todolist.md` 進度。

## 檔案變動
- `app.py`: 新增 20+ 行計分邏輯與 UI 渲染程式碼。
- `spec.md`: 新增計分系統規格。
- `README.md`: 完整重寫內容，包含功能描述與執行方式。
- `report.md`: 記錄本次任務成果。

## 結論
計分系統已成功整合至遊戲主迴圈中，玩家可以即時看到由於存活時間增加而獲得的高倍率得分，顯著提升了遊戲的可玩性與挑戰趣味度。
